Remote Desktop Tip from Darrell Norton

Thursday, February 5, 2004 by Jeffrey Randow
Darrell Norton over at dotnetjunkies posted a useful tip for those of you who frequently use Windows 2003 Remote Desktop Sessions: I have found at least one answer to this problem! In Windows Server 2003 (only), there is a new group policy setting called Restrict Terminal Services Users to a Single Remote Session. In the Group Policy snap-in, it is located under (Local Computer Policy/Computer Configuration/Administrative Templates/Windows Components/Terminal Services). Set the state to Enabled, and people won’t be able to have two connections. Read More
